## Часть 2. Схема подключения

### 2.1 Питание

```
USB (5В)
  └── TP4056 ──── LiPo 3.7В
                      │
          ┌───────────┼──────────────┐
          │           │              │
     AMS1117-3.3   MT3608        напрямую
          │        Vout=28В      не нужно
        3.3В           │
   (ESP32, WM8960)  V_BELL
```

**Настройка MT3608 перед подключением:**
1. Подать питание от LiPo без нагрузки
2. Мультиметром на VOUT+ / VOUT−
3. Крутить подстроечник до показания **28.0В**
4. Отключить питание

**Подтяжка EN MT3608:**
```
GND ── [10 кОм] ──┬── EN (площадка резистора 222 на плате)
                  │
      ESP32 GPIO13 ── [100 Ом] ──┘
```

### 2.2 Драйвер звонка

Спектр-3 (ТА-11320) использует **одну катушку МТ** (электромагнит).
Достаточно одного BS170 — пульсирующий ток на частоте 25 Гц.

```
28В
 └── [Катушка МТ] ──┬── Drain BS170
                    │   Gate  ── [100 Ом] ── GPIO27
                    │   Source ── GND
                 1N4148 (катод → 28В, анод → Drain)
```

Точки подключения на плате Спектр-3: контакты катушки МТ (ХТ8 и К по схеме).
Трансформатор Т и транзисторы VT1/VT2 (КТ315Б) из схемы не используются.

### 2.3 WM8960

| Пин WM8960 | Куда |
|---|---|
| VIN | 3.3В |
| SPKVDD | 3.3В |
| AVDD | 3.3В |
| GND | GND |
| EN | 3.3В |
| BCLK | ESP32 GPIO26 |
| DLRC | ESP32 GPIO25 |
| ALRC | ESP32 GPIO25 |
| DDAT | ESP32 GPIO17 |
| ADAT | ESP32 GPIO34 |
| SCL | ESP32 GPIO22 |
| SDA | ESP32 GPIO21 |
| HPL + HPR | Оба на + динамика трубки |
| OUT3 | На − динамика трубки |
| MICB | [2.2 кОм] → Vcc электретного капсюля |
| LIN1 | Сигнал электретного капсюля |
| LIN2 | GND |

### 2.4 Дисковый номеронабиратель

```
3.3В ── [10 кОм] ──┬── [10 нФ] ── GND
                   │
               GPIO33 (DIAL_PULSE)
                   │
               НД-контакт диска
                   │
                  GND

3.3В ── [10 кОм] ──┬── GPIO35 (DIAL_ACTIVE)
                   │
               НО-контакт диска
                   │
                  GND
```

### 2.5 Рычажный переключатель

```
3.3В ── [10 кОм] ──┬── [100 нФ] ── GND
                   │
               GPIO32 (HOOK_SWITCH)
                   │
            Hook switch (NC в покое)
                   │
                  GND
```

### 2.6 Микрофон (замена угольного)

Угольный капсюль требует ~48В — заменить электретным.
Электретный капсюль встаёт в то же гнездо трубки.

```
WM8960 MICB ── [2.2 кОм] ──┬── Vcc капсюля (+)
                            │
               WM8960 LIN1 ─┤── Сигнал капсюля
               WM8960 LIN2 ── GND капсюля (−)
```

## Часть 3. Прошивка

### 3.1 Установка ESP-IDF

1. Скачать инсталлятор: `https://dl.espressif.com/dl/esp-idf/`
   - Выбрать **Online installer**, версия **5.2** или новее
2. Запустить `.exe`, дождаться завершения (~10 минут)
3. На рабочем столе появится ярлык **ESP-IDF 5.x CMD**

### 3.2 Сборка проекта

Открыть **ESP-IDF 5.x CMD**:

```cmd
cd C:\Users\denibeer\Documents\VSCode\retro
idf.py set-target esp32
idf.py build
```

Первая сборка занимает 5–10 минут.

### 3.3 Прошивка

Подключить ESP32 DevKit по USB. Найти порт в Диспетчере устройств
(Порты COM и LPT → Silicon Labs CP210x или CH340).

```cmd
idf.py -p COM3 flash monitor
```

Заменить `COM3` на актуальный порт.

При запуске монитора в терминале должны появиться строки:
```
I (xxx) WM8960: init OK
I (xxx) MAIN: Retro Phone ready
```

### 3.4 Повторная прошивка

Если ESP32 не входит в режим прошивки автоматически:
1. Зажать кнопку **BOOT** на плате
2. Нажать и отпустить **EN** (Reset)
3. Отпустить **BOOT**
4. Запустить `idf.py -p COMx flash`

## Часть 4. Первое сопряжение Bluetooth

1. Включить питание устройства
2. На смартфоне: Настройки → Bluetooth → Поиск
3. В списке выбрать **"Retro Phone"**
4. Подтвердить PIN: **0000**
5. Убедиться что устройство разрешено для звонков

После сопряжения ESP32 запомнит телефон в NVS.
При следующем включении подключение произойдёт автоматически.

**Сброс привязки:** набрать **000** на дисковом номеронабирателе
при поднятой трубке и отсутствии BT-соединения.
Сброс срабатывает сразу после третьей цифры — ждать тайм-аута не нужно.
Подтверждение — короткий сигнал звонка (300 мс).

## Часть 5. Калибровка регистров WM8960

Все регистры изменяются в функции `wm8960_init()` в файле `main/audio.c`.
После каждого изменения — пересборка и перепрошивка: `idf.py -p COMx flash`.

### 5.1 Громкость в динамике трубки (Headphone out)

Регистры R2 (левый) и R3 (правый). Текущее значение: `0x170` (~−6 дБ).

| Значение | Уровень |
|---|---|
| `0x1FF` | +6 дБ (максимум) |
| `0x179` | 0 дБ |
| `0x170` | −6 дБ (по умолчанию) |
| `0x160` | −12 дБ |
| `0x100` | −73 дБ (почти тишина) |

Регулировка в `audio.c`:
```c
wm_write(2, 0x170);  // увеличить → 0x179, уменьшить → 0x160
wm_write(3, 0x170);  // оба канала одинаково
```

Советский динамик очень чувствительный — начинать с `0x160` и повышать.

### 5.2 Усиление микрофона (PGA)

Регистр R0 (левый вход). Биты [5:0] — усиление PGA от −17.25 дБ до +30 дБ,
шаг 0.75 дБ.

| Значение бит [5:0] | Усиление |
|---|---|
| `0x3F` | +30 дБ |
| `0x2F` | +17.25 дБ |
| `0x17` | 0 дБ (по умолчанию) |
| `0x00` | −17.25 дБ |

Полное значение регистра с битом IPVU (обновление): `0x100 | (gain & 0x3F)`.

```c
wm_write(0, 0x12F);  // +17.25 дБ, IPVU=1
```

Если собеседник плохо слышит — увеличить. Если фонит — уменьшить.

### 5.3 Усиление ADC (цифровой)

Регистр R21 (левый ADC). Диапазон: −97 дБ … 0 дБ, шаг 0.5 дБ.
0 дБ = `0x1C3` (с битом ADCVU).

```c
wm_write(21, 0x1C3);  // 0 дБ
wm_write(21, 0x1D3);  // +8 дБ (осторожно с перегрузкой)
```

### 5.4 Порядок калибровки микрофона

1. Позвонить на устройство, взять трубку
2. Попросить собеседника оценить качество звука
3. Если тихо: увеличить R0 на шаг (0x117 → 0x11F → 0x127...)
4. Если шумно/фонит: уменьшить R0 или R21
5. Повторить до приемлемого результата

### 5.5 Порядок калибровки динамика

1. Поднять трубку (должен быть гудок 425 Гц)
2. Отрегулировать R2/R3 до комфортной громкости гудка
3. Совершить тестовый звонок, оценить громкость разговора
4. При необходимости скорректировать

### 5.6 Настройка тайм-аутов дискового набора

Два независимых тайм-аута. Первый — между импульсами одной цифры,
второй — пауза после последней цифры, по истечении которой номер отправляется.

**Тайм-аут между импульсами одной цифры** (`main/config.h`):

```c
// Если пропускаются импульсы (цифры набираются неверно):
#define PULSE_MIN_MS     30   // уменьшить с 40

// Если две цифры сливаются в одну:
#define DIGIT_TIMEOUT_MS  350  // увеличить с 280

// Если одна цифра разбивается на две:
#define DIGIT_TIMEOUT_MS  200  // уменьшить с 280
```

**Тайм-аут завершения набора** (`main/state_machine.c`):

```c
#define DIAL_COMPLETE_MS  3000  // по умолчанию 3 секунды
```

После того как набрана последняя цифра и диск вернулся в исходное положение,
устройство ждёт `DIAL_COMPLETE_MS` миллисекунд. Если новых цифр нет — звонит.

| Значение | Когда использовать |
|---|---|
| 2000 мс | Опытный пользователь, быстрый набор |
| **3000 мс** | **По умолчанию, оптимально** |
| 4000 мс | Если номер часто обрывается до конца набора |

Примеры поддерживаемых номеров:
- `112` — экстренные службы
- `7XXXXXXXXXX` — мобильный (11 цифр)
- `XXXXXXX` — городской (7 цифр)
- любая другая длина — работает без ограничений

## Часть 6. Диагностика

### Монитор логов

```cmd
idf.py -p COM3 monitor
```

| Лог | Значение |
|---|---|
| `WM8960: init OK` | Кодек найден и инициализирован |
| `HFP: conn state: 3` | BT подключён (state=SLC_CONNECTED) |
| `SM: 0 -> 1` | Трубка поднята, переход в DIAL_TONE |
| `SM: 1 -> 2` | Начало набора |
| `HFP: paired: OK` | Сопряжение прошло успешно |
| `WM8960: reg 0xXX write failed` | Проверить I2C соединение |

### Частые проблемы

| Проблема | Причина | Решение |
|---|---|---|
| Нет гудка | WM8960 не инициализирован | Проверить провода SDA/SCL, адрес 0x1A |
| Звонок не звонит | MT3608 не выдаёт 28В | Перемерить выход, проверить BS170 |
| Диск не набирает | Нет импульсов на GPIO33 | Прозвонить контакт НД диска |
| BT не находится | ESP32 не в режиме поиска | Переприошить, проверить монитор |
| Эхо в разговоре | Громкость динамика слишком высокая | Уменьшить R2/R3 |
